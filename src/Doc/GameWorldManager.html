Class: GameWorldManager

GameWorldManager

new GameWorldManager(game, spaceBetweenWalls, platformSpriteWidth, wallSpriteHalfWidth, obstacleSpriteWidth, floorSpriteWidth, totalWalls, numberOfDoubleWalls, totalPlatforms, platformsPerRoom, totalObstacles, obstaclesPerRoom)

Class that controls the positioning, creation(Phaser based so it sets up rendering too), reseting and freezing of the World's Walls, Obstacles, Platforms and floors it randomly generates all of the WorldObjects positions based off of the concept of considering each space between two walls as a "Room"
Parameters:
Name Type Description
game Phaser.Game link to the currently active Phaser Game object
spaceBetweenWalls number the distance you wish to space the walls apart
platformSpriteWidth number size of the platform sprites width in pixels
wallSpriteHalfWidth number size of the wall sprites width halfed in pixels
obstacleSpriteWidth number size of the obstacle sprites width in pixels
floorSpriteWidth number size of the floor sprites width in pixels
totalWalls number total number of walls we wish in the game
numberOfDoubleWalls number this is how many "double walls" we wish which are walls that have a top and bottom with a small space between this gets deducted from the total walls so if we had 8 walls total and 2 double walls, we'd only have 6 set of walls two would be doubled up
totalPlatforms number total number of platforms we wish in the game
platformsPerRoom number number of platforms we want between each set of walls
totalObstacles number total number of obstacles we wish in the game
obstaclesPerRoom number number of obstacles we want between each set of walls

Methods

create()

Function that creates all the worlds objects as WorldObject's which creates Phaser Images and adds them to a Phaser Group allowing them to be displayed on screen

floorAndRoofPlacement(floorMin)

Function that positions the roof and floor sprites, not random just positions them one after the other
Parameters:
Name Type Description
floorMin number place to position the floor tiles, roof tiles are positioned at 0 (it should be a low value in phaser lower is higher)

freezeWorld()

Function that "freezes" all the currently held WorldObjects in GameWorldManager by setting there x velocity to 0

Function that randomizes the obstacle positions based off all of the GameWorldManagers passed in obstacle values
Parameters:
Name Type Description
topPosition number place to position the Obstacles on the top row (it should be a low value in phaser lower is higher)
bottomPosition number place to position the Obstacles on the bottom row (it should be a high value in phaser higher is lower)

Function that randomizes the platforms positions based off all of the GameWorldManagers passed in platform values
Parameters:
Name Type Description
maxHeight number the maximum height to position the platforms (it should be a low value in phaser lower is higher)
minHeight number the minimum height to position the platforms (it should be a high value in phaser higher is lower)

positionSave()

Function that saves all of the the GameWorldManagers WorldObjects positions to the internal position variables if we don't do this then when the states change the Groups that contain the WorldObject's internal image get deleted and we lose the position. We can still use its original position but we need there last known position for displaying them on screen through different states

reset()

Function that resets all the GameWorldManagers WorldObjects by destroying all the currently held group objects. And then re-randomizes there positions and call create to recreate them.

update(playerVelocityX)

Function that updates all the WorldObjects Phaser Group's updates there x position constantly incase of speed boosts and repositions all of them when they go off screen
Parameters:
Name Type Description
playerVelocityX number value that dictates the speed the world moves towards the player at should be the players current velocity on the X

wallRandomizer(topPosition, bottomPosition)

Function that randomizes and places the wall positions based off all of the GameWorldManagers passed in wall values, slightly less random than the other functions as the Obstacles and Platforms are based from the walls positions on the X
Parameters:
Name Type Description
topPosition number place to position the walls that are on the roof or highest point (it should be a low value in phaser lower is higher)
bottomPosition number place to position the Walls that are on the floor or lowest point (it should be a high value in phaser higher is lower)
