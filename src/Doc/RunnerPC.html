Class: RunnerPC

RunnerPC

new RunnerPC(posX, posY, scaleX, scaleY, rot, gravity, runSpeed, boostSpeed, animatedSprite, spriteFps, game)

RunnerPC is an object intended to be the players character(PC)
Parameters:
Name Type Description
posX number position of RunnerPC the x plane
posY number position of RunnerPC on the y plane, positive is down
scaleX number width scale of RunnerPC sprite
scaleY number height scale of RunnerPC sprite
rot number rotation of RunnerPC
gravity number amount of downward force applied to the RunnerPC
runSpeed number speed the character regularly moves at without boost
boostSpeed number speed the character moves at when boost is pressed
animatedSprite string string that's a key of an image loaded in via Phasers load.image function
spriteFps number the frames per second that the animated sprite should iterate through its animations at
game Phaser.Game phaser game object to allow access to game specific functions

separate(player, collidee)

Function that can be called separately anywhere you wish, however it's mainly for use in conjunction with the Phaser collide function as a callback function, so that whenever a Phaser object collides with the player it calls this function. Essentially all this does is seperate the two colliding objects on the Y axis so that they're still colliding (just touching) but no longer half way inside each other.
Parameters:
Name Type Description
player Phaser.sprite a sprite that denotes the player character it gets removed from the collidee object
collidee Phaser.sprite a sprite that denotes the obstacle the player sprite is colliding with dictates if a character is on the ground or not and thus able to invert the gravity
